# DC Garbage Collection Reform - Data Directory

## Coordinate Reference System (CRS)

**All spatial data uses EPSG:2248** (NAD83 / Maryland State Plane, US feet)

This CRS is chosen because:
- Native unit is feet (matches walking distance requirements: 250/500/750 ft)
- Accurate for the DC/Maryland region
- Minimizes distortion for distance calculations

## Data Sources

### Raw Data (`raw/`)

All raw data is fetched from the **DC Open Data Portal** via ArcGIS REST Services.
Run `python src/fetch_dc_data.py` to (re-)generate these files.

| File | Description | Source URL | Update Frequency | Committed? |
|------|-------------|------------|------------------|------------|
| `dc_streets.parquet` | Street centerlines | [DC GIS Transportation](https://maps2.dcgis.dc.gov/dcgis/rest/services/DCGIS_DATA/Transportation_WebMercator/MapServer/170) | Ongoing | No — fetch via script |
| `dc_blocks.parquet` | Census Blocks (2020) | [DC GIS Demographics](https://maps2.dcgis.dc.gov/dcgis/rest/services/DCGIS_DATA/Demographic_WebMercator/MapServer/8) | Decennial | Yes |
| `dc_addresses.parquet` | Address points | [DC GIS Location](https://maps2.dcgis.dc.gov/dcgis/rest/services/DCGIS_DATA/Location_WebMercator/MapServer/0) | Ongoing | Yes |

> **Note:** `dc_streets.parquet` is not committed to the repository because it is not
> consumed by any current analysis script (container placement uses address points only).
> It can be regenerated at any time with `python src/fetch_dc_data.py`.

### Processed Data (`processed/`)

| File | Description | Generated By |
|------|-------------|--------------|
| `container_locations.parquet` | Candidate container placement sites (18,247 containers) | `analysis/container_placement.py` |
| `block_stats.parquet` | Per-block distance statistics and threshold violation flags | `analysis/container_placement.py` |
| `placement_summary.json` | Aggregate walk-distance compliance by ward and threshold (250/500/750 ft) | `analysis/container_placement.py` |
| `capacity_analysis.json` | Capacity model results with growth scenario (+25%) and minimum pickup frequencies | `analysis/capacity_model.py` |
| `capacity_comparison.csv` | Human-readable current vs shared system capacity comparison table | `analysis/capacity_model.py` |
| `cost_analysis.json` | Capital and operating cost breakdown with parking impact | `analysis/cost_model.py` |
| `cost_comparison.csv` | Per-household and citywide cost comparison table | `analysis/cost_model.py` |

## Reproduction

To regenerate raw data:
```bash
python src/fetch_dc_data.py
```

The script is fully automated and requires no manual downloads.
