# DC Garbage Collection Reform - Data Directory

## Coordinate Reference System (CRS)

**All spatial data uses EPSG:2248** (NAD83 / Maryland State Plane, US feet)

This CRS is chosen because:
- Native unit is feet (matches walking distance requirements: 250/500/750 ft)
- Accurate for the DC/Maryland region
- Minimizes distortion for distance calculations

## Data Sources

### Raw Data (`raw/`)

All raw data is fetched from the **DC Open Data Portal** via ArcGIS REST Services.

| File | Description | Source URL | Update Frequency |
|------|-------------|------------|------------------|
| `dc_streets.parquet` | Street centerlines | [DC GIS Transportation](https://maps2.dcgis.dc.gov/dcgis/rest/services/DCGIS_DATA/Transportation_WebMercator/MapServer/170) | Ongoing |
| `dc_blocks.parquet` | Census Blocks (2020) | [DC GIS Demographics](https://maps2.dcgis.dc.gov/dcgis/rest/services/DCGIS_DATA/Demographic_WebMercator/MapServer/8) | Decennial |
| `dc_addresses.parquet` | Address points | [DC GIS Location](https://maps2.dcgis.dc.gov/dcgis/rest/services/DCGIS_DATA/Location_WebMercator/MapServer/0) | Ongoing |

### Processed Data (`processed/`)

| File | Description | Generated By |
|------|-------------|--------------|
| `container_locations.parquet` | Candidate container placement sites | `analysis/container_placement.py` |

## Reproduction

To regenerate raw data:
```bash
python src/fetch_dc_data.py
```

The script is fully automated and requires no manual downloads.
